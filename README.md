# Automated City Traffic Simulator

This project simulates a smart traffic grid where vehicles and traffic lights interact autonomously. It heavily tests your ability to manage shared resources (intersections) without causing deadlocks.

**Directory Structure Concept:**
Same as your template, utilizing `TrafficFacade.h`, `IntersectionObserver.h`, `VehicleFactory.h`, and `CentralDispatchLogger.h`.

```txt
    CityTrafficProject/
    ├── CMakeLists.txt
    ├── include/
    │   ├── CentralDispatchLogger.h    (Singleton)
    │   ├── IVehicle.h                 (Product Interfaces)
    │   ├── VehicleFactory.h           (Abstract Factory / Factory Method)
    │   ├── TrafficLight.h             (Observer Pattern - Subject)
    │   ├── IIntersectionObserver.h    (Observer Pattern - Interface)
    │   └── CityTrafficFacade.h        (Facade)
    ├── src/
    │   ├── CentralDispatchLogger.cpp
    │   ├── VehicleFactory.cpp
    │   ├── TrafficLight.cpp
    │   ├── CityTrafficFacade.cpp
    │   └── main.cpp
    └── build/                         (Generated by Ninja)
```


**Development Roadmap:**

* **Part 1: The Core Foundation (Singleton & Concurrency)**
* **Goal:** Implement a thread-safe `CentralDispatchLogger`.
* **Tasks:** Write a logger to track accidents, traffic light changes, and vehicle spawns. Ensure `std::mutex` prevents garbled text when dozens of vehicles log their positions simultaneously.


* **Part 2: The Infrastructure Ecosystem (Abstract Factory)**
* **Goal:** Generate the city's actors.
* **Tasks:** Create an `IVehicleFactory` to spawn concrete classes like `CivilianCar`, `PublicBus`, and `Ambulance`. Create an `IInfrastructureFactory` to generate `TrafficLight` and `SpeedCamera` objects.


* **Part 3: The Event System (Observer, Concurrency & Condition Variables)**
* **Goal:** Make vehicles react to lights and emergency vehicles.
* **Tasks:** Make `TrafficLight` the Subject and `Vehicles` the Observers. The lights run on independent `std::thread` loops, changing colors and calling `notify()`. Vehicles must pause their movement threads when notified of a red light. Use `std::condition_variable` to pause and wake up vehicle threads efficiently.


* **Part 4: The Command Center (Facade)**
* **Goal:** Orchestrate the simulation.
* **Tasks:** Build a `CityTrafficFacade` with methods like `initializeGrid(int intersections)` and `triggerRushHour()`. `main.cpp` simply calls these and watches the multithreaded simulation run.
